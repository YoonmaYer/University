파일시스템의 종류와 이해

FAT16, FAT32(MS-DOS) File Allocation Table

EXT계열 (EXT1, 2..3.. 현재는 4버전) Extended File System

윈도우즈 NT 계열(윈도우 2000~윈도우11)


파일시스템의 문제점

데이터의 중복, 종속성(묶여있음)
데이터간의 불일치성, 다수 사용자를 위한 동시제어(ex. 동시접근) 제공 x

보안기능 미흡


데이터베이스

여러 사용자나 응용 프로그램이 '공유'하고 '동시에 접근(동시제어)' 가능한 데이터의 '집합'



DBMS(DataBase Management System)

데이터베이스 관리 시스템(MySQL, ORACLE...)
데이터베이스를 관리 운영하는 소프트웨어



데이터베이스의 특징

데이터의 무결성(중요): 데이터베이스 안의 데이터는 어떤 경로를 통해 들어왔든 오류가 있어서는 안 됨
데이터의 독립성: 의존적인 관계가 아니라 독립적인 관계임
보안
데이터 중복 최소화: 중복 방지
응용 프로그램 제작 및 수정 용이: 쿼리를 통해 제작 용이
데이터의 안정성 향상


DBMS의 분류

1. 계층형 DBMS
각 계층이 트리 형태. 1 : N 관계
한번 구축하면 변경하기 까다로움
접근의 유연성이 부족함
요즘 안씀.

2. 망형 DBMS
1 : 1, 1 : N, N : N 관계 지원, 효과적이고 빠른 데이터 추출 가능
계층형보다는 낫지만 요즘 안씀

3. 관계형 DBMS(Relation)
모든 데이터는 '테이블'에 저장 - 중요
대용량 데이터를 체계적으로 관리할 수 있음 
데이터의 무결성이 잘 보장된다
시스템 자원을 많이 차지하여 시스템이 전반적으로 느려지는 단점 있음



SQL(Structured Query Language)

데이터베이스를 조작(생성,삭제, 수정 등)하는 언어
SQL은 ANSI/ISO(국제표준) 표준을 준수하기 때문에 DBMS이 변경되어도 그대로 사용 가능

SQL특징

DBMS 제작 회사와 독립적
다른 시스템과의 이식성이 좋음
표준이 계속 발전함
대화식 언어(질의)
클라이언트/서버 구조 지원

SQL종류

DDL(Data Definition Language): 데이터 정의어, 관계형 데이터베이스 테이블을 생성, 변경, 삭제하는 것으로 데이터를 저장할 구조를 저장하는 언어

DML(Data Manipulation Language): 데이터 조작어, 데이터 구조가 DDL로 정의되면 테이블에 데이터를 입력, 수정, 삭제, 조회한다.

DCL(Data Control Language): 데이터 제어어, 데이터베이스 사용자에게 권한을 부여하거나 회수


CRUD(Create, Read, Update, Delete)
