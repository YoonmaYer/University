데이터베이스 시스템 구조

1단계: 외부단계, 사용자 개인의 견해

2단계: 개념단계, 조직의 전체의 견해(개인의 견해들이 종합됨)

3단계: 내부단계, 물리적 저장장치의 입장에서 보는 저장장치의 견해


스키마

1단계: 외부 스키마, 개별 사용자, 여러 형태 존재, 서브 스키마라고도 불림

2단계: 개념 스키마, 조직 전체, 단 하나의 개념 스키마만 존재, 스키마라고 불림

3단계: 내부 스키마, 저장장치 입장에서 전체 DB가 저장되는 방법을 명세한 것, 하나의 내부 스키마만 존재

사상

1단계: 외부/개념 사상(응용 인터페이스),논리적 데이터 독립성 제공

2단계: 개념/내부 사상(저장 인터페이스), 물리적 데이터 독립성 제공

3단계: 내부/장치 사상(장치 인터페이스), 내부 스키마와 물리적인 장치간의 대응 관계 정의

데이터 독립성

'논리적' 데이터 독립성: 기존 응용 프로그램에 영향 주지 않고 논리적 구조 변경 가능
'물리적' 데이터 독립성: 응용 프로그램과 논리적 구조에 영향 x, 물리적 구조 변경 가능
 
데이터 모델 구성요소

데이터 구조: DB에서 표현될 대상과 그들 간의 관계에 대한 명세, static

연산: 데이터를 처리하는 작업에 대한 명세, dynamic

제약조건: 무결한(타입에 맞는 적절한) 데이터 저장을 위한 데이터 조작 한계에 대한 규정

데이터 모델의 종류

개념적 데이터 모델: ER모델(트리, 다이어그램으로 관계도를 나타냄), Entity-Relationship

논리적 데이터 모델: 테이블로 관계도를 나타냄, 모델(계층, 네트워크, 관계)

*계층: 트리
네트워크: 그래프 
관계: 테이블(열, 행)


개체: 변수 같은 느낌
속성(attrybute): 개체의 특성, 속성값은 개체의 특성을 현실화 한 것.
식별자

카디널리티: 관계를 나타내는 상대적인 개념(1:1 ~ n:n), 기수, 대응 수, 튜플의 개수

*릴레이션 관련 용어

문자: CHAR, VARCHAR
숫자: INT, FLOAT
날짜/시간: DATE,TIME

릴레이션의 특성:

1. 유일성: 하나의 릴레이션에는 중복된 튜플 x
2. 튜플의 무순서성: 하나의 릴레이션에는 튜플 간의 순서 상관 없음
3. 속성의 무순서성: 하나의 릴레이션에는 속성 순서 상관 없음
4. 속성의 원자성: 모든 속성값은 원자값만 사용할 수 있음
